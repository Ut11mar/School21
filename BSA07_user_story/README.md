# BSA07 — Пользовательские истории (User Stories)

## Описание проекта

Проект направлен на освоение техники написания пользовательских историй (User Story) как инструмента формализации требований. В ходе работы составлялись истории для различных ролей, формулировались критерии приёмки и разрабатывались тестовые сценарии. Дополнительно проводилось командное обсуждение и согласование написанных историй.

---

## Что было сделано

- `ex00_BRB_us_manager.docx`  
  Пользовательские истории для роли менеджера в домене барбершопа. К каждой истории приложены либо критерии приёмки, либо тестовый сценарий, позволяющий проверить выполнение требования.

- `ex01_BRB_us_roles.docx`  
  Истории для остальных ролей системы барбершопа — клиента и мастера. По каждой роли проработаны несколько сценариев с критериями приёмки и тестовыми кейсами.

- `ex02_bs_us.docx`  
  Документ по итогам командного обсуждения пользовательских историй. Зафиксированы замечания, спорные формулировки и принятые решения по доработке историй.

---

## Итоги

В рамках проекта отработана методика составления пользовательских историй: от корректной формулировки по шаблону до написания критериев приёмки и тестовых сценариев. Совместное обсуждение историй с коллегами позволило выявить неточности и улучшить качество требований. Навык написания User Stories востребован при работе в Agile-командах и при составлении Product Backlog.
