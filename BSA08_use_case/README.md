# BSA08 — Use Cases (Сценарии использования)

## Описание проекта

Проект по разработке Use Case-документации для системы онлайн-записи барбершопа. Система рассматривается как «чёрный ящик». Описаны взаимодействия трёх ролей — менеджера, клиента и мастера — с оформлением атрибутов UC, основных и альтернативных потоков, связей include и extend.

---

## Что было сделано

- `ex00_BRB_UC.docx`
  **UC-01** — Актор: **менеджер**. Описание (без потоков): ведение расписания услуг мастеров. Зафиксированы атрибуты, стейкхолдеры, предусловия, триггер, минимальные гарантии.

- `ex01_BRB_UC.docx`
  **UC-01** — Потоки: основной (добавление расписания) + 4 альтернативных (некорректные данные, корректировка, удаление, шаблон графика).

- `ex02_BRB_UC.docx`
  **UC-02-BOOK / UC-02-INC / UC-02-EXT** — Актор: **клиент**. Описание (без потоков): бронирование услуги мастера. Связи: include (указать контакты) и extend (отправить напоминание).

- `ex03_BRB_UC.docx`
  **UC-02-BOOK / UC-02-INC / UC-02-EXT** — Потоки: клиент выбирает услугу → мастера → слот → подтверждает. Альтернативы: незарегистрированный клиент, занятый слот, некорректные данные.

- `ex04_BRB_UC.docx`
  **UC-04-VIEW / UC-05-REV** — Актор: **мастер**. Описание (без потоков): просмотр расписания и записей клиентов. Связь include с UC-AUTH. Мастер видит только свои записи.

- `ex05_BRB_UC.docx`
  **UC-AUTH / UC-04-VIEW / UC-05-REV** — Потоки: аутентификация (логин → проверка → доступ по роли), просмотр записей мастера за период. UC-AUTH переиспользуется через include.

---

## Итоги

- описал UC с полным набором атрибутов: идентификатор, актор, цель, триггер, предусловия, гарантии, постусловия;
- расписал основные и альтернативные потоки для трёх ролей;
- применил связи **include** и **extend** на практике;
- выстроил структуру: описание UC (чётные файлы) → потоки (нечётные файлы).
