# BSA08 — Use Cases (Сценарии использования)

## Описание проекта

Проект направлен на освоение методологии Use Case — подхода к фиксации требований через сценарии взаимодействия пользователей с системой. В рамках работы были разобраны три роли: клиент, менеджер и мастер. Для каждой роли прописаны ключевые атрибуты сценария (идентификатор, актор, цель, триггер, предусловия, гарантии, постусловия), описаны основные и альтернативные потоки, а также установлены связи между сценариями через механизмы include и extend.

---

## Что было сделано

- `ex00_srs_UC.docx`  
  Разработан Use Case для роли **менеджера**: поддержание актуальности расписания услуг. Зафиксированы цель, триггер, актор, предусловия и гарантии. Подробно расписаны основной поток действий и альтернативные ветки — взаимодействие менеджера с расписанием и перечнем услуг.

- `ex02_srs_UC.docx`  
  Составлен сценарий для роли **клиента**: запись на услугу через систему. Описаны связи между Use Case-ами (include / extend), шаги основного сценария и варианты отклонений от стандартного пути.

- `ex04_srs_UC.docx`  
  Проработаны сценарии для роли **мастера**: просмотр записей и отзывов. Определены действия, связи между UC, ограничения доступа и возможные альтернативные пути выполнения.

---

## Итоги

В процессе выполнения проекта удалось:

- разобраться со структурой Use Case и научиться корректно заполнять все его атрибуты: актор, цель, триггер, предусловия, гарантии и постусловия;
- составлять основные и альтернативные потоки для разных ролей пользователей;
- применять связи **include** и **extend** для выстраивания зависимостей между сценариями;
- моделировать поведение системы с учётом ролевой модели (клиент, менеджер, мастер);
- оценить, как Use Case помогает команде разработки говорить с заказчиком на одном языке.
